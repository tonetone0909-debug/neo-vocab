# 학생 코드 로그인 설정 (Google Sheet + Apps Script)

학생별 코드로만 앱을 쓰게 하고, 관리자가 코드를 삭제하면 해당 학생이 더이상 못 쓰게 하는 기능이에요.
**아래 설정을 1회 해두고, 마지막에 배포 URL을 `assets/config.js`에 넣으면 켜져요.** (URL을 비워두면 인증 없이 기존처럼 동작해요.)

## 1) 구글 스프레드시트 만들기
1. https://sheets.google.com 에서 새 시트 생성.
2. 왼쪽 아래 시트 탭 이름을 **`codes`** 로 변경.
3. (선택) 1행에 머리글 없이 바로 데이터로 써도 돼요. 열 구성: **A=코드, B=라벨(학생 이름 등), C=생성일**.

## 2) Apps Script 붙여넣기
1. 시트 상단 메뉴 **확장 프로그램 → Apps Script**.
2. 기본 `Code.gs` 내용을 지우고, `build/apps_script_auth.gs` 파일 내용을 **전부 복사해 붙여넣기** → 저장.

## 3) 관리자 코드 설정
1. Apps Script 왼쪽 **⚙ 프로젝트 설정 → 스크립트 속성 → 속성 추가**.
2. 속성 이름 **`ADMIN_CODE`**, 값에 원하는 **관리자 코드**(예: `admin-7788`) 입력 → 저장.

## 4) 웹앱으로 배포
1. Apps Script 오른쪽 위 **배포 → 새 배포**.
2. 유형 선택(톱니) → **웹 앱**.
3. 설명 입력, **실행 계정: 나**, **액세스 권한: 모든 사용자** 선택 → 배포.
4. 권한 승인(본인 구글 계정). 배포 후 나오는 **웹앱 URL**(`https://script.google.com/macros/s/.../exec`)을 복사.

## 5) 앱에 URL 연결
1. `6. neo-toefl/assets/config.js` 열기.
2. `url: ""` 안에 복사한 `/exec` URL을 붙여넣기 → 저장.
   ```js
   window.NEO_AUTH = { url: "https://script.google.com/macros/s/AKfy.../exec" };
   ```
3. `sw.js`의 `CACHE` 버전을 한 칸 올리고(예: v18→v19) 다시 배포/새로고침하면 적용돼요.

## 사용 방법
- **학생**: 앱 첫 화면(로그인)에서 코드 입력 → 통과하면 사용. 앱을 열 때마다 온라인으로 코드 유효성 확인(오프라인이면 사용 불가).
- **관리자**: 로그인 화면에 관리자 코드 입력 → 관리자 페이지에서 코드 발급/삭제. (또는 구글 시트 `codes`에서 직접 행 추가/삭제해도 됨.)
- **차단**: 관리자가 코드를 삭제하면, 그 학생은 다음 실행(온라인 검증) 때 자동 로그아웃·차단돼요.

## 코드 추가/삭제(관리자)
- **앱 관리자 페이지**: 로그인에서 관리자 코드 입력 → `admin.html` 에서 추가/삭제.
- **시트 직접**: `codes` 시트 A열에 코드 추가, 삭제는 행 삭제.

## 진도 동기화 업데이트 (기능 추가 시)
스크립트에 **진도 동기화**(같은 코드면 어느 기기에서나 진도 동일)가 추가됐어요. 이미 한 번 배포했다면, 새 코드로 **업데이트 배포**만 하면 돼요:
1. 구글 시트 → **확장 프로그램 → Apps Script** → 기존 코드 전체 지우고 `build/apps_script_auth.gs` 최신 내용 붙여넣기 → 저장.
2. `progress` 시트는 **자동 생성**돼요(따로 안 만들어도 됨). A=코드, B=진도, C=수정일시.
3. 오른쪽 위 **배포 → 배포 관리** → 기존 배포 옆 **연필(편집)** → 버전 **"새 버전"** 선택 → **배포**. (➡️ `/exec` URL은 그대로 유지돼요.)
4. 앱은 이미 동기화 코드가 들어있어서, URL만 그대로면 자동으로 작동해요.
- 참고: 진도는 구글 시트 한 칸(약 5만 자)에 저장돼요. 일반적인 학습량은 충분하고, 아주 많이 쌓이면 추후 분할이 필요할 수 있어요.

## 학생 진도 보기 (관리자 페이지)
관리자 페이지에 **학생 진도** 탭이 추가됐어요(학생별 외운 단어·학습률·마지막 접속·덱별 진도). 최신 `build/apps_script_auth.gs`를 붙여넣고 위와 같이 **"새 버전" 배포**만 다시 하면 작동해요(`getallprog` 엔드포인트 포함, `/exec` URL 그대로).

**이 배포를 하기 전까지는 Writing 진도가 서버에 안 올라가고 관리자에도 안 보여요** (어휘는 그대로 잘 됩니다).

1. 구글 시트 → **확장 프로그램 → Apps Script** → 기존 코드 전체 지우고 `build/apps_script_auth.gs` 최신 내용 붙여넣기 → 저장.
2. **배포 → 배포 관리** → 기존 배포 옆 **연필(편집)** → 버전 **"새 버전"** → **배포**. (`/exec` URL 그대로 유지 → `config.js` 손댈 필요 없음)
3. `progress_w1` 시트는 **자동 생성**돼요(따로 안 만들어도 됨). A=코드, B=Writing 진도, C=수정일시.

**왜 시트를 나눴나요?** 시트 한 칸이 5만 자 한도라 두 앱 진도를 한 칸에 못 넣어요.

| 앱 | `ns` | 시트 |
|---|---|---|
| 어휘 | (안 보냄) | `progress` ← 기존 그대로 |
| Writing | `w1` | `progress_w1` ← 새로 생김 |

`ns`를 안 보내면 예전과 똑같이 `progress` 시트로 가요. 그래서 **이미 학생 폰에 깔린 앱도 그대로 동작**하고, 기존 어휘 진도는 건드리지 않아요.

## 참고/주의
- 간단 코드 모델이라 코드가 곧 출입증이에요(아이디/비번 아님). 코드는 학생마다 다르게, 추측 어렵게(예: `neo-3f9k2`) 발급하세요.
- 관리자 코드는 URL 쿼리로 전송되니 외부에 노출되지 않게 관리하세요.
- 오프라인 차단이 기본이라, 인터넷이 없으면 학습이 안 돼요(요청 사항). 원하면 추후 "오프라인 임시 허용"으로 완화 가능.
- CORS 오류가 나면 배포 **액세스 권한이 '모든 사용자'** 인지 확인하세요.
